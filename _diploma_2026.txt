Взаимодействие между клиентским приложением и сервером осуществляется с помощью API-запросов. API (Application Programming Interface) представляет собой механизм взаимодействия между различными программными компонентами, который позволяет обмениваться данными и выполнять различные операции над информацией.В разработанном приложении API используется для передачи информации о состоянии пользователя между клиентским приложением и сервером. Клиентское приложение отправляет запросы на сервер для добавления новых записей о настроении, получения существующих данных, а также выполнения различных операций с информацией. Сервер принимает запросы, выполняет необходимую обработку данных и возвращает результат клиенту.Для передачи данных между клиентом и сервером используется протокол HTTP. Этот протокол является стандартом взаимодействия в веб-приложениях и широко применяется для обмена информацией между программными системами. В рамках разработанного приложения используются различные типы HTTP-запросов, каждый из которых выполняет определенную функцию [5].Основными типами запросов, используемых в системе, являются POST-запросы для отправки данных на сервер, GET-запросы для получения информации из базы данных и DELETE-запросы для удаления записей.Данные между клиентом и сервером передаются в формате JSON (JavaScript Object Notation).
